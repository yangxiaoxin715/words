# 手机闪卡工具界面 SOP

适用场景：手机端高频闪卡、刷题、听读等 3-5 分钟工具。

## 1. 先分两层

- 首页负责看进度、选任务、进工具。
- 操作页只负责当前一轮练习。
- 不把数据、工具、账号信息和当前操作混在同一个首屏。

## 2. 首页只留高频判断

- 今日进度
- 当前阶段进度
- 主按钮
- 必要的次按钮

低频动作进折叠工具区：导入、导出、账号、编号、设置。

## 3. 操作页保持沉浸

- 顶部：返回、任务名、当前进度。
- 中间：当前卡片。
- 底部：反馈按钮。
- 反馈按钮位置稳定，不让用户每张卡都重新找。
- 底部反馈按钮要按拇指操作设计，不能沿用普通工具按钮尺寸。
- 底部反馈按钮不要贴底，至少留 40px，带底部安全区的设备再叠加 24px。

## 4. 自动模式少给选择

- 用户会：点屏幕表示准备核对。
- 用户不会：超时自动记为新朋友。
- 核对后只给必要按钮，例如老朋友、有点眼熟。
- 3 秒窗口用轻量进度条提示，不用大号数字倒计时，避免制造催促感。

## 5. 手机端稳定性

- 刷词页的倒计时、进度提示优先用 `transform` 动画，不用 `width` 动画。
- 高频切换的区域保持占位稳定，不要每张卡都 `display:none` 再显示。
- 词卡主区域加绘制隔离，例如 `contain: layout paint`，减少微信 WebView 整块重绘。
- 单词字号可以大，但要给长词保留换行或收缩空间，不能挤出卡片。
- 自动模式音频播放如果卡住，要在兜底发音后继续启动倒计时，不能让流程停在当前单词。

## 6. 页面检查清单

- 手机首屏能不能一眼看到开始按钮？
- 刷词时有没有账号、导出、工具信息干扰？
- 反馈按钮是否不用滚动就能点到？
- 自动模式是否能感到时间在走，又不会被倒计时数字压迫？
- 返回后是否能看到进度变化？
- 低频功能是否被收起，但还能找到？
