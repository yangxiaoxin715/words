# 单词音频预加载设计

日期：2026-06-21

## 目标

解决单词卡片出现后，音频因网络请求延迟一到数秒才开始播放，挤占三秒答题时间的问题。

验收口径：

- 每轮开始前，先准备本轮全部单词的音频。
- 音频全部准备完成后才展示第一张单词卡。
- 每张卡片出现时，音频立即播放，同时启动完整的三秒答题倒计时。
- 三秒答题时间不包含音频下载等待。
- 超时后展示中文一秒，再进入下一个单词。
- 用户在三秒内点击卡片或作答时，沿用现有交互。

## 方案

### 每轮音频准备

用户点击“开始”或“继续”后，页面进入音频准备状态。程序为本轮单词分别创建音频对象并等待可播放状态。

本轮所有音频都准备完成后，进入测试并展示第一张卡片。这样网络等待集中发生在开始前，不进入任何单词的答题计时。

### 播放与计时

每个单词使用已经准备好的独立音频对象。

展示卡片时：

1. 将对应音频播放位置重置到开头。
2. 立即播放音频。
3. 同时启动三秒倒计时。

不再在展示卡片时替换远程音频地址。

### 加载失败

如果任一音频加载失败：

- 不进入测试。
- 停留在准备状态。
- 显示加载失败提示和“重新加载”按钮。
- 重新加载时重新准备本轮全部音频，避免混用状态不确定的缓存。

### 生命周期

- 开始新一轮时清空上一轮音频缓存。
- 退出测试时停止当前音频并清空缓存。
- 重新加载时取消旧的准备批次，旧批次即使稍后返回也不能启动测试。

## 界面

复用测试页，在准备阶段隐藏单词卡片、倒计时和答题按钮，显示：

- 正常状态：“正在准备音频…”
- 失败状态：“音频加载失败，请重新加载”
- 失败操作：“重新加载”

不新增独立页面。

## 测试

自动化测试至少覆盖：

- 音频未全部准备完成时，不展示卡片，不启动倒计时。
- 全部音频准备完成后，只启动一次测试。
- 单个音频失败时，不启动测试并显示重试状态。
- 重试成功后可以正常进入测试。
- 展示卡片时播放对应缓存音频并启动三秒倒计时。
- 超时后中文仍展示一秒。

浏览器验证：

- 首轮开始前能看到准备状态。
- 第一张及后续卡片出现时音频无明显网络等待。
- 每张卡片拥有完整三秒答题时间。
- 网络失败时不会出现无声倒计时。
