# 和单词交朋友网页版

英语词汇闪卡与学习记录工具。

## 本地运行

```bash
cd /Users/yangming/Desktop/英语开窍2000词/2000词英语开窍营/word-hunter-web
../platform/.venv/bin/python app.py
```

打开：

```text
http://127.0.0.1:8000/
```

如果 8000 被占用，可以临时换端口：

```bash
../platform/.venv/bin/python -m uvicorn app:app --host 127.0.0.1 --port 8010
```

后台产品数据与账号创建：

```text
http://127.0.0.1:8000/admin.html
```

默认管理口令：

```text
dev-admin-key
```

生产环境请设置：

```bash
export WORD_HUNTER_ADMIN_KEY="换成强口令"
export WORD_HUNTER_DB="/data/word_hunter.db"
export WORD_HUNTER_AUDIO_LIBRARY="/data/word-hunter/audio-library"
export WORD_HUNTER_AUDIO_CACHE="/data/word-hunter/audio-cache"
```

## 私有账号

第一版不是公开注册工具。用户必须使用你发放的学习编号和密码登录。

后台创建账号：

1. 打开 `/admin.html`
2. 输入后台口令
3. 在“创建学习账号”里填写学习编号和初始密码；也可以留空自动生成
4. 把学习编号和初始密码发给用户

后台账号管理：

- 重置密码：用于用户忘记密码
- 停用/启用：用于暂停某个账号访问
- 删除：用于清理本地测试账号，会同时删除这个账号的学习记录
- 词明细：查看这个账号每个词的状态、见过次数、最后学习时间
- 导出：导出这个账号的新朋友、有点眼熟、老朋友词表，方便后续生成个性化故事

用户端登录：

```text
学习编号：00001
密码：后台创建时生成或填写的密码
```

学习数据跟账号走，不跟设备走。换手机或电脑后，用户用同一个学习编号和密码登录，会继续使用同一份云端学习记录。

本轮进度也跟账号走。用户刷到一半退出后，下次登录会继续上一轮剩下的词；点“再来一轮”才会生成新的一轮词。

阶段捕获进度按 200 个词一组计算：

- 第一组：1-200
- 第二组：201-400
- 用户点“老朋友”的词计入“已捕获”
- 页面显示当前阶段捕获进度，例如 `第一组 187 / 200`
- “冲刺剩下”只刷当前阶段还没有捕获的词
- 当天连续两次成为“新朋友”的词，会在主流程里排到后面，减少反复卡住
- “新朋友复习”只刷当前阶段的“新朋友”和“有点眼熟”
- 当前阶段完成后，用户手动点击“进入第二组”

页面布局原则：

- 手机优先，首页和刷词页分开
- 首页只放关键数据、阶段进度、模式切换和“开始”
- 刷词页只服务当前一轮：返回、标题、进度、单词卡、反馈按钮
- 反馈按钮贴在刷词页底部，减少手指找按钮
- 导入词包和导出数据收进“工具”，默认折叠

用户端导出：

- 登录后点击“导出数据”
- 下载文件名类似 `word-hunter-00001-2026-07-12.csv`
- CSV 可用 Excel 打开
- 文件包含学习编号、导出时间、词状态、单词、中文、见过次数、最后学习时间
- 用户可以把这个文件发给你，后续交付网站再导入使用

导入词包：

- 用户可以导入 CSV 词包
- CSV 格式为 `word,meaning`
- 一个账号只保留一个当前导入词包
- 新导入会覆盖旧词包
- 每个词包最多 100 个词
- 页面只显示“导入词包”，不写故事或课程

安全边界：
- 密码不明文存储
- 学习接口需要登录令牌
- 不开放自助注册
- 不采集真实姓名、手机号、学校、班级、照片、录音等隐私数据

## 单词音频

前端固定请求：

```text
/api/audio/{word_id}?voice=us
```

后端按这个顺序找音频：

1. 自有音频库：`WORD_HUNTER_AUDIO_LIBRARY/us/apple.mp3`
2. 有道音频缓存：`WORD_HUNTER_AUDIO_CACHE/youdao/us/apple.mp3`
3. 有道词典美式发音：`dict.youdao.com/dictvoice`

如果后端音频不可用，前端会退回浏览器自带英文发音。

后期替换成自有音频库时，不需要改前端。把生成好的 MP3 放到自有音频库目录即可。文件名规则：

```text
Apple -> apple.mp3
good morning -> good-morning.mp3
T-shirt -> t-shirt.mp3
```

## 功能边界

保留：
- 单词闪卡
- 单词释义
- 单词点击播放
- 自动模式
- 手动模式
- 今日见面
- 新朋友
- 有点眼熟
- 老朋友
- 连续见面
- 阶段捕获进度
- 冲刺剩余词
- 新朋友复习
- 导入词包
- 刷导入词包
- 本轮进度和剩余数量
- 用户端导出数据
- 学习编号和密码登录
- 后台创建学习账号
- 云端同步学习记录
- 后台聚合产品数据

自动模式：
- 点“开始”后自动播放英文单词。
- 3 秒内点卡片，会先显示中文，再选择“老朋友 / 有点眼熟”。
- 3 秒没点，会先亮出中文 1 秒，再记为“新朋友”并进入下一个词。

不做：
- 手机号登录
- 公开注册
- 真实姓名
- 学校、班级
- 课程购买
- 在线培训
- 直播课
- 老师点评
- 故事音频课程包

## 部署建议

国内备案网页建议使用腾讯云或阿里云轻量应用服务器。

推荐：
- Ubuntu LTS
- Python 3.11+
- Nginx 反向代理
- systemd 托管 `uvicorn`
- SQLite 数据库放在持久化目录
- 域名、服务器、备案在同一云厂商处理

项目已包含上线需要的词库文件：

```text
words-data.js
```

生产部署模板：

```text
.env.example
deploy/word-hunter.service.example
deploy/nginx.conf.example
deploy/README.md
```
